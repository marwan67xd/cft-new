# i18n Configuration Validation

This document ensures the i18n configuration is properly set up and prevents ENOENT errors.

## Required Files

- ✅ `i18n/locales/en.json` - English translations
- ✅ `i18n/locales/ar.json` - Arabic translations
- ✅ `i18n/i18n.config.ts` - Vue I18n configuration
- ✅ `nuxt.config.ts` - Nuxt configuration with i18n module

## Validation

Run the validation script before building:

```bash
npm run validate:i18n
```

This script will:
1. Check if `i18n/locales/` directory exists
2. Verify all required locale files exist
3. Validate JSON syntax in locale files
4. Confirm `nuxt.config.ts` references match locale files

## Adding New Locales

When adding a new locale:

1. Create the locale file: `i18n/locales/{code}.json`
2. Add the locale configuration to `nuxt.config.ts`:
   ```ts
   {
     code: '{code}',
     iso: '{iso-code}',
     name: '{Display Name}',
     file: '{code}.json',
     dir: '{ltr|rtl}'
   }
   ```
3. Run validation: `npm run validate:i18n`

## Nuxt 4 Compatibility

This configuration is compatible with:
- Nuxt 4.3.1+
- @nuxtjs/i18n 10.2.3+
- Vue I18n 11+

## Troubleshooting

If you encounter ENOENT errors:

1. Run `npm run validate:i18n` to identify missing files
2. Ensure `langDir: 'locales'` matches your directory structure
3. Verify all locale files are valid JSON
4. Check that `nuxt.config.ts` locale `file` properties match actual filenames
